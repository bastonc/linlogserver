Simple logger for Linux based radio station

Shortly review

After installation

Run LinuxLog from application menu you system. If it's first start LinuxLog - you will see Hello window.

In this window - please entry you callsign and press "GO" LinuxLog close and show again in full mode

After runing you'l see some windows:

  • Telnet cluster window. This window have a very simply functional: You click into slot - Linux Log do autofill callfield and setup transceiver to frequency spot
  • Log window - list all QSOs
  • Search repeats window - it's window listing repeats qso with callsign which the entry to Main form
  • Main window - it's main form application. If close this window - LinuxLog is closed.
    This window has a many entry field:
    • Callsign field - enter callsign here
    • RSTr/RSTs - enter RST reciev and sent here
    • Name - enter name of correspondent here
    • QTH - enter QTH of correspondent here
    • Comment - in this field you can enter comments about this station. ATTENTION This field don't supported ADI standart
    • Band- select BAND for you QSO in this list. If using TCI - it's field in autofill mode
    • Mode - select MODE for you QSO. If using TCI - it's field in autofill mode

    Also this window has a digital clocks with local and global (GMT) time. Your call sign display in this window too. If using TCI protocol in this window display frequency.

  • Internet search window. This window display image of correspondent

Diplom managment functional

Diplom manager is called from menu item - "Other"


This manager has some fields:

  • Name diploma - input field. In this field you need enter name diplom programm
  • Select color for telnet highlight - button by select color for highlight in cluster, if cluster reciv call from call list for this diplom
  • How many points do you need - input field. How many points do you have to score enter in this field
  • Repeats - list of conditions for possible repeats
  • Table with columns - call and scores. Enter into strings all special callsign and how many points added this station at complited QSO
  • Button "Add rows" - adding to table new string (if needed)
  • Use only prefix. If you need to consider only prefixes in the diploma program - check it
  • Button "Cancel" - close manager without saving
  • Button "Create diploma" - saving diplom settings and running filters for cluster
After saving the diploma program, filters are started for the cluster and the main journal.
If the cluster received the callsign included in special call list - the line in the cluster window is highlighted.

You may using functional the diploma manager for filtering callsigns from spotline. Why not :-)

If you have completed QSO with a station from the list of special callsigns - QSO is added into the main base and to the log file of your diploma program. If you scored the required number of points - LinuxLog will notify you of this. By result performance the diploma program, you can export the ADI file. In this file included QSO from special call list only.

"Manager diplom programms" is constantly developing and may change

How to install

  1. Download installer
  2. Set executable bit ("chmod -X name installer file" or use GUI - Mouse Right Button -> "Premissions" -> checkbox "Executable")
  3. Run installer (recomended execute in terminal - if you have errors - please take screenshot errors and send to my email - bastonsv@gmail.com)

After installation run LinLog from menu DE

LinLog and installer tested into Ubuntu, Kubuntu, Mint Cinnamon, Fedora Workstation, Fedora KDE (spins)



How enter QSO into base

Enter QSO begins from input callsign in main window. Inputing in field can by hands, or autofill from cluster.
After input callsign - press TAB or Enter
   If you press TAB - cursor moves to RSTr field. In this moment LinuxLog run search photo for this radiostation on qrz.com server. Search image process working in parallel thread so image can showing with delay. In RSTr/s fields enter relevant units
  If you press Enter - QSO saved in base. After this manupulation all fields is setup clean.
In other fields entry information as needed Important to remember press Enter in any fields = saving QSO to base


Menu

Main window LinuxLog has a menu string



  • Menu "Settings" - it's a main menu application has some tabs:

    First what you need know about general menu (settings) - it's three button at the bottom of window.

    Cancel - this button closing menu window without any changes
    Apply - this button applying all changes but don't saving changes
    Save and exit - this button applying all changes and saving it into settings


    • General:

      Fields:
      - You Callsign - enter you callsign here :)
      - Window color - press this button for change color window
      - Text color - press this button for change text color in LinuxLog
      - Form color - for change color in background tables and all fields press this button



    • Cluster

      Fields:
      - Cluster host and port - ip adress and port (if required) telnet cluster. By default it's dxfun.com 8000
      You can use other telnet clusters, but you need callibrate (see down).
      - Filter BAND chekbox - enabling filtering by BAND. All bands need enter in field "bands in m.". separate bands by comma (example: 20,40,30,160)
      - Filter SPOT chekbox - enabling filtering spot by prefix. You need enter prefixes spots showing in cluster window (example: EA,UR,IT,JA)
      - Filter SPOTTER checkbox - enabling filtering spotter by prefix. You need enter prefixes separated by comma spotters showing in cluster window (example: EA,UR,IT,JA)
      - Start calibrate - this button starting calibrate cluster. After pressed on this button, LinuxLog awaiting string from telnet server (it' take a few seconds, maybe minutes it depends on cluster activity), and you need indicate relevant element for callsign and frequency from list.
      If you using cluster by default (dxfun.com) - you don't need the process of cluster callibrate





    • TCI

      Fields:
      - TCI Enable checkbox - enable protocol TCI for drive you transciever (SunSDR)
      - Fields TCI host and TCI port - you need entry IP and PORT you TCI device. By default setting localhost:40001. If you using local transciever (connected to computer where running LinuxLog) you don't need change this fields




    • Log file Tab

      Fields:
      - Button import / export Log file
      You can import adi file into base LinuxLog or export you logbook from LinuxLog to .adi file






    • Services Tab

      Fields:
      - Checkbox to enabled remote services (in this time moment - only eqsl.cc)
      - Fields for login and password from you account in service. If you activated eqsl service - after add qso in base, LinuxLog send eqsl for correspondent in automatically mode. ATTENTION LinuxLog don't recieving you eQSL card, because processing qsl card - it task not for logger!
